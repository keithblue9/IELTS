# Ascent IELTS — PRD

## Original problem statement
Personal IELTS app modeled on Fluently — AI tutor (speaking/writing/listening/reading), configurable target band, dynamic questions, post-submit teaching, pain-points tracking, PIN-only personal login, PWA installable, no Emergent watermark.

## Stack
- Frontend: React 19 + Tailwind + shadcn/ui + framer-motion + recharts (axios + sonner)
- Backend: FastAPI + Motor (MongoDB) + Pydantic v2 + bcrypt + python-jose (JWT)
- LLM: Claude Sonnet 4.5 via emergentintegrations (EMERGENT_LLM_KEY)
- Audio: OpenAI Whisper (STT) + OpenAI TTS (9 voices)
- PWA: manifest.json + service-worker.js + icons (192/512/180 apple-touch)

## Core requirements (locked)
- **PIN auth** (6 digits, default `123456`, changeable in Settings); single owner user auto-seeded
- **Dynamic content**: every listening/reading/writing prompt is freshly generated by Claude on demand (no stale repeats)
- **Post-submit teaching**: each listening/reading question has an explanation (quote from source + IELTS technique tip); writing has annotated_feedback + band-8 model answer
- **Pain points dashboard**: aggregates speaking/writing criteria + listening band; surfaces 3 weakest with custom tips
- **PWA**: installable on mobile/desktop, safe-area-aware, bottom tab bar for mobile
- **No Emergent watermark**: removed from index.html + CSS belt-and-braces hide
